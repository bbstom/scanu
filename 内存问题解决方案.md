# Node.js 内存溢出问题解决方案

## 问题描述

错误信息：
```
FATAL ERROR: Ineffective mark-compacts near heap limit Allocation failed - JavaScript heap out of memory
```

这是 Node.js 内存不足导致的错误，在 Windows 开发环境中比较常见。

## 已实施的解决方案

### 1. 修改 package.json（已完成）

在启动脚本中增加内存限制：

```json
{
  "scripts": {
    "dev": "node --max-old-space-size=4096 node_modules/next/dist/bin/next dev",
    "build": "node --max-old-space-size=4096 node_modules/next/dist/bin/next build"
  }
}
```

### 2. 创建 .npmrc 文件（已完成）

在项目根目录创建 `.npmrc` 文件：

```
node-options=--max-old-space-size=4096
```

## 使用方法

现在可以正常启动项目：

```bash
npm run dev
```

内存限制已从默认的 ~1.5GB 增加到 4GB。

## 其他解决方案

### 方案 1：使用 cross-env（跨平台）

如果上述方案不生效，可以安装 cross-env：

```bash
npm install --save-dev cross-env
```

修改 package.json：

```json
{
  "scripts": {
    "dev": "cross-env NODE_OPTIONS=--max-old-space-size=4096 next dev",
    "build": "cross-env NODE_OPTIONS=--max-old-space-size=4096 next build"
  }
}
```

### 方案 2：设置环境变量（Windows）

临时设置（当前终端有效）：

```cmd
set NODE_OPTIONS=--max-old-space-size=4096
npm run dev
```

PowerShell：

```powershell
$env:NODE_OPTIONS="--max-old-space-size=4096"
npm run dev
```

永久设置（系统环境变量）：

1. 右键"此电脑" → 属性
2. 高级系统设置 → 环境变量
3. 新建系统变量：
   - 变量名：`NODE_OPTIONS`
   - 变量值：`--max-old-space-size=4096`
4. 重启终端

### 方案 3：优化项目配置

如果内存问题持续，可以优化 Next.js 配置：

修改 `next.config.mjs`：

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  images: {
    domains: ['localhost'],
  },
  output: 'standalone',
  
  // 优化配置
  swcMinify: true,
  compiler: {
    removeConsole: process.env.NODE_ENV === 'production',
  },
  
  // 减少内存使用
  experimental: {
    optimizeCss: true,
  },
};

export default nextConfig;
```

### 方案 4：清理缓存

有时清理缓存可以解决问题：

```bash
# 删除 .next 目录
rm -rf .next

# 删除 node_modules
rm -rf node_modules

# 清理 npm 缓存
npm cache clean --force

# 重新安装
npm install

# 启动
npm run dev
```

## 内存限制说明

- `--max-old-space-size=2048`：2GB（最小推荐）
- `--max-old-space-size=4096`：4GB（推荐）
- `--max-old-space-size=8192`：8GB（大型项目）

根据你的电脑内存选择合适的值。

## 预防措施

1. **定期重启开发服务器**
   - 长时间运行可能导致内存泄漏
   - 建议每隔几小时重启一次

2. **关闭不必要的进程**
   - 确保没有多个 Node.js 进程在运行
   - 关闭其他占用内存的应用

3. **使用生产模式构建**
   - 开发模式占用更多内存
   - 测试时可以使用 `npm run build && npm start`

4. **优化代码**
   - 避免大量的 console.log
   - 减少不必要的依赖
   - 使用动态导入（lazy loading）

## 如果问题仍然存在

1. **检查系统内存**
   - 确保电脑有足够的可用内存
   - 关闭其他占用内存的程序

2. **升级 Node.js**
   - 使用最新的 LTS 版本
   - 新版本有更好的内存管理

3. **检查依赖**
   - 某些依赖可能导致内存泄漏
   - 尝试更新或替换问题依赖

4. **使用 Docker**
   - 在 Docker 容器中运行可以更好地控制资源
   - 但这需要额外的配置
