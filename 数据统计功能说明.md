# 数据统计功能说明

## 功能概述

系统提供了完整的数据统计和监控功能，帮助管理员了解系统使用情况、性能指标和用户行为。

### 3. 数据导出

```http
GET /api/admin/export?format=csv&chain=eth&limit=1000
Authorization: Bearer <token>
```

**参数**:
- `format`: 导出格式
  - `json`: JSON 格式（默认）
  - `csv`: CSV 格式
- `chain`: 链类型（可选）
  - `eth`, `btc`, `trx`
- `limit`: 导出数量（默认 1000）

**CSV 格式**:

```

**JSON 格式**:

#### 获取缓存统计

```

**响应**:
```json

      },
      "btc": {
        "count": 234,

      },
      "trx": {
        "count": 345,
        "keys": [...]
      },
      "ratelimit": {
        "count": 199,
        "keys": [...]
      }
    },
    "memoryUsed": "45.6M"
  }
}
```

#### 清除缓存

```http
DELETE /api/admin/cache?type=prefix&pattern=eth
Authorization: Bearer <token>
```

**参数**:
- `type`: 清除类型
  - `key`: 清除单个键
  - `prefix`: 清除指定前缀的所有键
  - `all`: 清除所有缓存
- `pattern`: 缓存键或前缀

**示例**:
```bash
# 清除所有以太坊缓存
DELETE /api/admin/cache?type=prefix&pattern=eth

# 清除所有限流记录
DELETE /api/admin/cache?type=prefix&pattern=ratelimit

# 清除所有缓存
DELETE /api/admin/cache?type=all

# 清除单个缓存
DELETE /api/admin/cache?type=key&pattern=eth:balance:0x742d35Cc...
```

## 统计指标说明

### 基础指标

| 指标 | 说明 |
|------|------|
| totalSearches | 总搜索次数（累计） |
| uniqueAddresses | 唯一地址数 |
| recentSearches | 指定时间范围内的搜索次数 |
| favoriteCount | 收藏地址数量 |

### 链统计

| 指标 | 说明 |
|------|------|
| totalSearches | 该链的总搜索次数 |
| uniqueAddresses | 该链的唯一地址数 |
| avgSearchCount | 平均搜索次数 |
| totalBalanceUSD | 总余额（美元） |

### 趋势数据

- **每日搜索趋势**: 最近 30 天每天的搜索次数和唯一地址数
- **标签统计**: 最常用的 20 个标签
- **搜索频率分布**: 按搜索次数分组统计

import { useEffect, useState } from 'react';

function AdminDashboard() {

### 2. 数据导出

导出搜索历史数据：

```typescript
async function exportData(format: 'json' | 'csv') {
  const response = await fetch(
    `/api/admin/export?format=${format}&limit=10000`,
    {
      headers: { 'Authorization': `Bearer ${token}` }
    }
  );

  if (format === 'csv') {
    const blob = await response.blob();
    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `search-history-${Date.now()}.csv`;
    a.click();
  } else {
    const data = await response.json();
    console.log('导出数据:', data);
  }
}
```

### 3. 缓存管理

管理系统缓存：

```typescript
async function clearCache(type: 'all' | 'prefix' | 'key', pattern?: string) {
  const url = new URL('/api/admin/cache', window.location.origin);
  url.searchParams.set('type', type);
  if (pattern) {
    url.searchParams.set('pattern', pattern);
  }

  const response = await fetch(url.toString(), {
    method: 'DELETE',
    headers: { 'Authorization': `Bearer ${token}` }
  });

  const result = await response.json();
  console.log(result.message);
}

// 使用示例
clearCache('prefix', 'eth');  // 清除以太坊缓存
clearCache('all');            // 清除所有缓存
```

### 4. 健康检查

定期检查系统健康状态：

```typescript
async function checkHealth() {
  const response = await fetch('/api/health');
  const health = await response.json();

  if (health.status !== 'healthy') {
    // 发送告警
    console.error('系统异常:', health);
    
    // 检查具体服务
    if (health.services.mongodb.status !== 'connected') {
      console.error('MongoDB 连接失败');
    }
    if (health.services.redis.status !== 'connected') {
      console.error('Redis 连接失败');
    }
  }

  return health;
}

// 每分钟检查一次
setInterval(checkHealth, 60 * 1000);
```

## 监控告警

### 建议监控的指标

1. **系统健康**
   - MongoDB 连接状态
   - Redis 连接状态
   - 内存使用率
   - 系统运行时间

2. **业务指标**

   - API 错误率

   - 缓存命中率

3. **资源使用**

   - Redis 内存使用
   - 服务器 CPU/内存

## 故障排查

### 问题 1: 统计数据不准确

**原因**:
- 缓存未更新
- 数据库索引缺失
- 聚合查询错误

**解决**:
```bash
# 清除统计缓存
curl -X DELETE -H "Authorization: Bearer $TOKEN" \
  "http://localhost:3000/api/admin/cache?type=prefix&pattern=stats"

# 重建索引
mongo crypto-wallet-explorer --eval "db.searchhistories.reIndex()"
```

### 问题 2: 导出数据失败

**原因**:
- 数据量过大
- 内存不足
- 超时

**解决**:
- 减少导出数量（limit 参数）
- 分批导出
- 增加服务器内存

### 问题 3: 健康检查失败

**原因**:
- MongoDB 连接断开
- Redis 连接断开
- 网络问题

**解决**:
```bash
# 检查 MongoDB
mongo --eval "db.adminCommand('ping')"

# 检查 Redis
redis-cli ping

# 重启服务
pm2 restart app
```
