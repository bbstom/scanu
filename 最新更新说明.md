## 快速开始

### 1. 启动项目
```bash
# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置 MongoDB 和 Redis

# 启动开发服务器
npm run dev
```

### 2. 访问后台
1. 访问 http://localhost:3000/admin
2. 首次访问会进入初始化页面
3. 设置管理员账号和密码
4. 登录后自动跳转到 API 配置页面

### 3. 配置 API
1. 在左侧菜单点击"API 配置"
2. 选择要配置的代币（ETH/BTC/TRX/价格）
3. 为每个平台配置：
   - 启用/禁用
   - 设置优先级（数字越小优先级越高）
   - 配置 API URL 和 Key
4. 可以添加新平台或删除现有平台
5. 点击"保存配置"立即生效

### 4. 其他配置
- **缓存配置**：设置余额和交易的缓存时间
- **限流配置**：设置 API 限流规则
- **网站配置**：设置网站名称、描述等信息
- **数据统计**：查看搜索统计和热门地址

## 核心特性

### 1. 多平台支持
每个代币可以配置多个 API 平台，系统会按优先级依次尝试：
- 优先级 1 的平台失败 → 自动切换到优先级 2
- 优先级 2 的平台失败 → 自动切换到优先级 3
- 以此类推，直到成功或所有平台都失败

### 2. 灵活配置
- 可以启用/禁用任意平台
- 可以调整平台优先级
- 可以添加自定义平台
- 配置立即生效，无需重启

### 3. 高可用性
- 多平台备份，单点故障不影响服务
- 自动切换，用户无感知
- 详细日志，便于排查问题

### 4. 缓存优化
- Redis 缓存系统
- 余额缓存 60 秒
- 交易缓存 120 秒
- 大幅提升响应速度

## 技术实现

### 配置管理
- 配置存储在 MongoDB 的 Settings 集合
- 每个代币的配置独立存储
- 配置键名：`API_CONFIG_ETH`、`API_CONFIG_BTC`、`API_CONFIG_TRX`、`API_CONFIG_PRICE`

### API 调用流程
```
1. 从数据库读取配置
2. 过滤启用的平台
3. 按优先级排序
4. 依次尝试调用
5. 成功则返回结果
6. 失败则尝试下一个
7. 所有平台都失败则返回错误
```

### 缓存机制
```
1. 检查 Redis 缓存
2. 缓存命中则直接返回
3. 缓存未命中则调用 API
4. API 成功则写入缓存
5. 设置过期时间
```

## 常见问题

### Q: 如何添加新的 API 平台？
A: 在对应的 API 配置页面，点击"添加平台"按钮，填写平台信息即可。

### Q: 配置修改后需要重启吗？
A: 不需要，配置修改后立即生效。

### Q: 如何查看 API 调用日志？
A: 查看服务器控制台输出，会显示详细的调用日志。

### Q: 所有平台都失败了怎么办？
A: 系统会返回错误信息，建议检查：
1. API Key 是否正确
2. API URL 是否可访问
3. 网络连接是否正常
4. API 是否有调用限制

### Q: 如何优化响应速度？
A: 
1. 配置 Redis 缓存
2. 将响应快的平台设置为高优先级
3. 禁用不稳定的平台
4. 使用付费 API 提高限额
