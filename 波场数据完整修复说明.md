## 修复内容

### 1. TRX 余额单位转换
**问题**: TronScan 和 TronGrid 返回的余额单位是 sun（1 TRX = 1,000,000 sun），但代码没有转换。

**修复**:
```typescript
// 修复前
balance: res.data.balance || 0

// 修复后
balance: (res.data.balance || 0) / 1e6
```

### 2. USDT 代币显示
**问题**: 代币数据实际上是正常返回的，但由于缓存中有旧数据导致不显示。

**解决方案**:
1. 清除 Redis 缓存
2. 重启服务器
3. 代币数据会正常显示

**验证**: 测试 API 返回了 470 个代币，包括 3 个 USDT 相关代币，总余额 562,727 USDT。

### 3. 交易记录字段完善
**问题**: 交易记录缺少 `type`、`result`、`tokenName`、`contractAddress` 等字段。

**修复**:
```typescript
// TRX 交易
{
  hash: tx.hash,
  from: tx.ownerAddress,
  to: tx.toAddress,
  value: (tx.amount / 1e6).toFixed(6),
  timestamp: tx.timestamp,
  type: "Transfer",                          // ✅ 新增
  result: tx.confirmed ? "SUCCESS" : "PENDING", // ✅ 新增
  tokenSymbol: "TRX",
  tokenName: "Tron",                         // ✅ 新增
  contractAddress: "",                       // ✅ 新增
}

// TRC-20 交易
{
  hash: tx.transaction_id,
  from: tx.from_address,
  to: tx.to_address,
  value: (tx.quant / Math.pow(10, tx.tokenInfo?.tokenDecimal || 6)).toFixed(6),
  timestamp: tx.block_ts,
  type: "Transfer",                          // ✅ 新增
  result: tx.finalResult || "SUCCESS",       // ✅ 新增
  tokenSymbol: tx.tokenInfo?.tokenAbbr || "Unknown",
  tokenName: tx.tokenInfo?.tokenName || "Unknown Token", // ✅ 新增
  contractAddress: tx.tokenInfo?.tokenId || "",          // ✅ 新增
  tokenType: "trc20",                        // ✅ 新增
}
```

### 4. 类型定义更新
**修复**: 在 `src/types/index.ts` 中添加缺少的字段：

## 常见问题

### Q: 为什么 USDT 还是不显示？
A: 最常见的原因是 Redis 缓存中有旧数据。解决方法：
1. 清除 Redis 缓存：`redis-cli` → `FLUSHDB`
2. 重启服务器
3. 重新查询

### Q: 如何验证 API 是否正常？
A: 访问测试端点：
```
http://localhost:3000/api/test/tron-direct?address=YOUR_ADDRESS
```
查看返回的 JSON 数据。

### Q: 交易记录中的地址为什么显示不对？
A: 已修复。确保：
1. 使用最新代码
2. 清除了缓存
3. 重启了服务器
