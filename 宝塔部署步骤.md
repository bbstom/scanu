# 宝塔面板部署步骤（内存优化版）

## 问题说明

你遇到的 `SIGBUS` 错误是因为构建时内存不足。我已经优化了配置，现在使用更少的内存。

---

## 快速部署（3步完成）

### 第1步：上传部署脚本

1. 在宝塔面板，点击 **文件**
2. 进入项目目录：`/www/wwwroot/scanu.cc/scanu`
3. 找到 `deploy-production.sh` 文件
4. 右键点击，选择 **权限**，设置为 `755`

### 第2步：运行部署脚本

1. 点击宝塔左侧 **终端**
2. 执行以下命令：

```bash
cd /www/wwwroot/scanu.cc/scanu
bash deploy-production.sh
```

脚本会自动：
- 检查系统资源
- 创建 Swap（如果需要）
- 清理旧文件
- 安装依赖
- 构建项目

### 第3步：启动应用

构建成功后，执行：

```bash
npm start
```

或使用 PM2（推荐）：

```bash
pm2 start npm --name crypto-app -- start
pm2 save
pm2 startup
```

## 手动部署（如果脚本失败）

### 1. 创建 Swap 空间

```bash
# 检查当前 Swap
free -h

# 如果 Swap 小于 1GB，创建 2GB Swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 设置开机自动挂载
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 验证
free -h
```

### 2. 清理并安装依赖

```bash
cd /www/wwwroot/scanu.cc/scanu

# 清理
rm -rf .next node_modules/.cache

# 安装依赖
npm ci
```

### 3. 构建项目（低内存模式）

```bash
# 方法 A: 使用最小内存（推荐）
NODE_OPTIONS='--max-old-space-size=512' npm run build

# 方法 B: 如果方法 A 失败，使用更小的内存
NODE_OPTIONS='--max-old-space-size=256' npm run build

# 方法 C: 如果还是失败，使用默认内存
npm run build
```

### 4. 启动应用

```bash
# 直接启动
npm start

# 或使用 PM2
pm2 start npm --name crypto-app -- start
pm2 save
```

## 配置反向代理（让外网访问）

### 1. 在宝塔面板添加站点

1. 点击 **网站**
2. 点击 **添加站点**
3. 填写信息：
   - 域名：`scanu.cc` 或 `www.scanu.cc`
   - 根目录：`/www/wwwroot/scanu.cc`
   - PHP版本：纯静态
4. 点击 **提交**

### 2. 配置反向代理

1. 找到刚创建的站点，点击 **设置**
2. 点击 **反向代理**
3. 点击 **添加反向代理**
4. 填写信息：
   - 代理名称：`crypto-app`
   - 目标URL：`http://127.0.0.1:3000`
   - 发送域名：`$host`
5. 点击 **提交**

### 3. 配置 SSL（可选但推荐）

1. 在站点设置中，点击 **SSL**
2. 选择 **Let's Encrypt**
3. 勾选你的域名
4. 点击 **申请**
5. 申请成功后，开启 **强制HTTPS**

## 环境变量配置

### 1. 复制环境变量文件

```bash
cd /www/wwwroot/scanu.cc/scanu
cp .env.example .env
```

### 2. 编辑 .env 文件

在宝塔面板：
1. 点击 **文件**
2. 找到 `.env` 文件
3. 右键点击，选择 **编辑**
4. 填写以下信息：

```env
# MongoDB 连接
MONGODB_URI=mongodb://localhost:27017/crypto-explorer

# Redis 连接
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

# 管理员密码（首次设置时使用）
ADMIN_PASSWORD=your-secure-password

# API 密钥（如果有）
BLOCKCHAIN_API_KEY=
ETHERSCAN_API_KEY=
TRONSCAN_API_KEY=

# 网站配置
NEXT_PUBLIC_SITE_URL=https://scanu.cc
```

## 常见问题

### Q1: 构建时还是内存不足怎么办？

**方案 A：在本地构建**

1. 在你的电脑上：
```bash
npm install
npm run build
```

2. 打包构建文件：
```bash
tar -czf build.tar.gz .next
```

3. 上传到服务器：
   - 在宝塔面板，点击 **文件**
   - 上传 `build.tar.gz`
   - 右键解压

4. 在服务器启动：
```bash
npm ci --production
npm start
```

**方案 B：使用更小的内存限制**

```bash
NODE_OPTIONS='--max-old-space-size=256' npm run build
```

### Q2: 如何查看应用日志？

```bash
# PM2 日志
pm2 logs crypto-app

# 或直接运行查看
npm start
```

### Q3: 如何重启应用？

```bash
# PM2
pm2 restart crypto-app

# 或停止后重新启动
pm2 stop crypto-app
pm2 start crypto-app
```

### Q4: 如何更新代码？

```bash
cd /www/wwwroot/scanu.cc/scanu

# 拉取最新代码（如果使用 Git）
git pull

# 或上传新文件后

# 重新构建
NODE_OPTIONS='--max-old-space-size=512' npm run build

# 重启应用
pm2 restart crypto-app
```

### Q5: 端口被占用怎么办？

```bash
# 查看占用 3000 端口的进程
lsof -i :3000

# 或
netstat -tlnp | grep 3000

# 杀死进程
kill -9 <PID>
```

## 性能优化建议

### 1. 启用 Gzip 压缩

在宝塔面板的站点设置中：
1. 点击 **配置文件**
2. 添加以下配置：

```nginx
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json;
```

### 2. 配置缓存

```nginx
location /_next/static {
    expires 365d;
    add_header Cache-Control "public, immutable";
}

location /static {
    expires 365d;
    add_header Cache-Control "public, immutable";
}
```

### 3. 限制请求大小

```nginx
client_max_body_size 10M;
```
