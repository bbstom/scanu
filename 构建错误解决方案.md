## 常见问题

### Q: 为什么构建需要这么多内存？
A: Next.js 构建过程需要编译、优化和打包所有页面和资源，这是一个内存密集型操作。

### Q: Swap 会影响性能吗？
A: Swap 主要用于构建时，运行时影响很小。而且我们设置了 `swappiness=10`，优先使用物理内存。

### Q: 可以在更小的服务器上运行吗？
A: 可以！构建可以在本地完成，服务器只需要运行构建好的应用，内存需求会小很多（512MB 也可以）。

### Q: 如何监控内存使用？
A: 使用 `htop` 或 `free -h` 命令，或在宝塔面板查看系统监控。

## 快速解决命令

```bash
# 一键解决（创建 Swap + 重新构建）
cd /www/wwwroot/scanu.cc/scanu && \
sudo fallocate -l 2G /swapfile && \
sudo chmod 600 /swapfile && \
sudo mkswap /swapfile && \
sudo swapon /swapfile && \
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab && \
rm -rf .next && \
node --max-old-space-size=2048 node_modules/next/dist/bin/next build
```
