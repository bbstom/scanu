# 数据准确性修复说明

## 修复日期
2024-03-01

## 问题描述

在后台系统重构后，发现以下问题：
1. 波场 (TRX) 余额数据不准确
2. USDT 代币无法正确显示
3. 交易记录数量限制未生效

## 问题原因

### 1. 余额单位转换错误
TronScan 和 TronGrid API 返回的余额单位是 **sun**（1 TRX = 1,000,000 sun），但代码中没有进行单位转换，导致余额显示错误。

**错误代码：**
```typescript
return {
  balance: res.data.balance || 0,  // 错误：直接使用 sun 单位
  // ...
};
```

**正确代码：**
```typescript
return {
  balance: (res.data.balance || 0) / 1e6,  // 正确：转换为 TRX
  // ...
};
```

### 2. 交易记录限制参数缺失
`getTrxTransactions` 函数缺少 `limit` 参数，导致无法控制返回的交易数量。

**错误代码：**
```typescript
export async function getTrxTransactions(address: string): Promise<TrxTransaction[]>
```

**正确代码：**
```typescript
export async function getTrxTransactions(address: string, limit: number = 50): Promise<TrxTransaction[]>
```

### 3. USDT 代币显示问题
USDT 代币数据实际上是正常返回的，但由于余额单位错误，导致整体数据显示异常。修复余额单位后，USDT 显示也恢复正常。

          return await getTransactionsFromTronScan(address, platform, limit);  // ✅ 传递 limit
        } else if (platform.name === "TronGrid" || platform.url.includes("trongrid")) {
          return await getTransactionsFromTronGrid(address, platform, limit);  // ✅ 传递 limit
        } else {

        }
      });
    },

}

      start: 0,
