# 后台设置页面优化说明

## 更新日期
2024-03-01

## 主要改进

### 1. 左侧菜单布局 🎨

将原来的顶部标签页改为左侧垂直菜单，提供更好的导航体验。

#### 布局特点
- **左侧固定菜单**: 宽度 256px，包含所有配置选项
- **右侧内容区**: 自适应宽度，显示当前选中的配置内容
- **图标导航**: 每个菜单项都有对应的图标，更直观
- **高亮显示**: 当前选中的菜单项有蓝色背景高亮

#### 菜单项
1. **API 配置** - 配置各区块链的 API 平台
2. **缓存配置** - 设置缓存时间
3. **限流配置** - 设置访问限制
4. **网站配置** - 自定义网站信息
5. **返回控制台** - 返回管理员主页
6. **数据统计** - 查看统计数据
7. **退出登录** - 退出管理员账号

### 2. 多平台 API 配置 🔧

每个区块链现在可以配置多个 API 平台，系统会按优先级自动切换。

#### 支持的区块链

**以太坊 (ETH)**
- Etherscan（默认优先级 1）
- Ethplorer（默认优先级 2）
- Blockscout（默认优先级 3）

**比特币 (BTC)**
- Blockchain.info（默认优先级 1）
- Blockchair（默认优先级 2）
- Blockcypher（默认优先级 3）

**波场 (TRX)**
- TronGrid（默认优先级 1）
- TronScan（默认优先级 2）

**价格查询**
- CoinGecko（默认优先级 1）

#### 平台配置项

每个平台包含以下配置：

1. **启用/禁用**: 复选框控制是否启用该平台
2. **平台名称**: 可自定义平台名称
3. **优先级**: 数字越小优先级越高（1 最高）
4. **API URL**: 平台的 API 地址
5. **API Key**: 可选，留空使用免费 API

#### 功能特性

- ✅ **添加平台**: 每个区块链可以添加自定义平台
- ✅ **删除平台**: 可以删除不需要的平台
- ✅ **优先级排序**: 系统按优先级顺序尝试调用
- ✅ **启用/禁用**: 灵活控制哪些平台可用
- ✅ **自动切换**: API 失败时自动切换到下一个平台

### 3. 配置存储方式

#### 数据结构

API 配置以 JSON 格式存储在数据库中：

```json
{
  "key": "API_CONFIGS",
  "value": "[{\"chainName\":\"以太坊 (ETH)\",\"platforms\":[...]}]",
  "description": "API 平台配置"
}
```

#### 配置示例

```json
[
  {
    "chainName": "以太坊 (ETH)",
    "platforms": [
      {
        "name": "Etherscan",
        "url": "https://api.etherscan.io/api",
        "apiKey": "YOUR_API_KEY",
        "enabled": true,
        "priority": 1
      },
      {
        "name": "Ethplorer",
        "url": "https://api.ethplorer.io",
        "apiKey": "",
        "enabled": true,
        "priority": 2
      }
    ]
  }
]
```

## 使用指南

### 配置 API 平台

1. 登录后台管理系统 `/admin`
2. 点击"系统设置"
3. 在左侧菜单选择"API 配置"
4. 为每个区块链配置平台：
   - 勾选启用/禁用复选框
   - 设置优先级（1-99）
   - 填写 API URL
   - 填写 API Key（可选）
5. 点击"保存所有设置"

### 添加自定义平台

1. 在对应区块链的配置区域
2. 点击"+ 添加平台"按钮
3. 填写平台信息：
   - 平台名称（如 "自定义节点"）
   - API URL
   - API Key（如果需要）
   - 设置优先级
4. 保存设置

### 删除平台

1. 找到要删除的平台
2. 点击右上角的"删除"按钮
3. 保存设置

### 调整优先级

1. 修改平台的优先级数字
2. 数字越小，优先级越高
3. 系统会按优先级从小到大依次尝试
4. 保存设置

#### 核心函数
- `loadSettings()`: 加载配置，包括 API 配置
- `handleSave()`: 保存所有配置到数据库
- `updateApiPlatform()`: 更新单个平台配置
- `addApiPlatform()`: 添加新平台
- `removeApiPlatform()`: 删除平台

### 数据类型

```typescript
interface ApiPlatform {
  name: string;
  url: string;
  apiKey: string;
  enabled: boolean;
  priority: number;
}

interface ChainApiConfig {
  chainName: string;
  platforms: ApiPlatform[];
}
```

### 后端集成

配置保存到 MongoDB 的 `settings` 集合中，key 为 `API_CONFIGS`。

后续需要更新区块链服务文件，从数据库读取配置并按优先级调用 API。

```typescript
// 伪代码示例
async function callApiWithFallback(chainType: string) {
  // 1. 从数据库读取 API_CONFIGS
  const configs = await getApiConfigs(chainType);
  
  // 2. 按优先级排序
  const sortedPlatforms = configs.platforms
    .filter(p => p.enabled)
    .sort((a, b) => a.priority - b.priority);
  
  // 3. 依次尝试调用
  for (const platform of sortedPlatforms) {
    try {
      const result = await callApi(platform);
      return result;
    } catch (error) {
      console.log(`${platform.name} 失败，尝试下一个...`);
      continue;
    }
  }
  
  throw new Error("所有 API 平台都失败");
}
```

## 配置说明

### 优先级规则

1. 数字越小，优先级越高
2. 优先级 1 会最先被调用
3. 如果优先级相同，按添加顺序调用
4. 禁用的平台会被跳过

### API Key 规则

1. 如果填写了 API Key，使用该 Key 调用
2. 如果留空，使用免费公共 API
3. 某些平台必须提供 API Key 才能使用

### 平台启用规则

1. 至少保留一个启用的平台
2. 所有平台都禁用时，使用默认免费 API
3. 建议保留 2-3 个备用平台
