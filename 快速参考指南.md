# 快速参考指南

## 项目启动

### 1. 安装依赖
```bash
npm install
```

### 2. 配置环境变量
```bash
cp .env.example .env
```

编辑 `.env` 文件：
```env
MONGODB_URI=mongodb://localhost:27017/crypto-wallet-explorer
REDIS_URL=redis://localhost:6379
```

### 3. 启动开发服务器
```bash
npm run dev
```

访问 http://localhost:3000

## 后台管理

### 首次设置
1. 访问 `/admin/setup`
2. 创建管理员账号
3. 登录后台

### 配置 API
1. 登录 `/admin`
2. 自动进入以太坊 API 配置
3. 配置各个代币的 API 平台
4. 保存配置

- `src/lib/apiConfig.ts` - API 配置管理
- `src/lib/settings.ts` - 配置读取

## API 端点

### 前端 API
- `GET /api/search` - 搜索地址
- `GET /api/history` - 搜索历史
- `GET /api/health` - 健康检查

### 后台 API
- `POST /api/admin/login` - 管理员登录
- `GET /api/admin/settings` - 获取配置
- `POST /api/admin/settings` - 保存配置
- `GET /api/admin/stats` - 数据统计

## 配置优先级

1. 环境变量 (.env)
2. 数据库配置 (后台设置)
3. 默认值 (代码)

## 缓存时间

- 余额: 60 秒
- 交易: 120 秒
- 代币: 120 秒
- 统计: 300 秒

## 限流配置

- 默认: 30 次/分钟
- 搜索: 20 次/分钟
- 可在后台调整

## 故障排查

### MongoDB 连接失败
```bash
# 检查 MongoDB 是否运行
mongosh

# 检查连接字符串
echo $MONGODB_URI
```

### Redis 连接失败
```bash
# 检查 Redis 是否运行
redis-cli ping

# 检查连接字符串
echo $REDIS_URL
```

### API 调用失败
1. 检查后台 API 配置
2. 查看控制台日志
3. 测试 API 平台可用性

### 内存不足
```bash
# 增加 Node.js 内存限制
node --max-old-space-size=4096 node_modules/next/dist/bin/next dev
```

## 性能优化

### 1. 启用 Redis
确保 Redis 正常运行，缓存命中率 > 90%

### 2. 配置多个 API 平台
每个代币至少配置 2-3 个备用平台

### 3. 调整缓存时间
根据实际需求在后台调整缓存时间

### 4. 使用 CDN
生产环境建议使用 CDN 加速静态资源

## 安全建议

### 1. 管理员密码
使用强密码，定期更换

### 2. API Keys
不要在代码中硬编码，使用环境变量或后台配置

### 3. 限流配置
根据服务器性能调整限流参数

### 4. HTTPS
生产环境必须使用 HTTPS

## 常见问题

### Q: 如何添加新的 API 平台？
A: 登录后台 → 对应代币配置 → 点击"+ 添加平台"

### Q: 如何调整缓存时间？
A: 登录后台 → 缓存配置 → 修改时间 → 保存

### Q: 如何查看数据统计？
A: 登录后台 → 数据统计

### Q: 如何修改网站名称？
A: 登录后台 → 网站配置 → 修改名称 → 保存

### Q: API 调用失败怎么办？
A: 系统会自动切换到备用平台，如果所有平台都失败，请检查网络和 API 配置
