```typescript
export async function getBalance(address: string) {
  return getCachedDataWithFallback(
    cacheKey,
    async () => {
      return await callApiWithFallback("CHAIN_TYPE", async (platform) => {
        // 根据平台选择不同的实现
        if (platform.name === "Platform1") {
          return await getBalanceFromPlatform1(address, platform);
        } else if (platform.name === "Platform2") {
          return await getBalanceFromPlatform2(address, platform);
        }
        // ...
      });
    },
    cacheTTL
  );
}
```

## 功能特点

### 1. 多平台支持

每个区块链都支持多个 API 平台：

**以太坊 (ETH)**:
- Etherscan
- Ethplorer
- Blockscout

**比特币 (BTC)**:
- Blockchain.info
- Blockchair
- Blockcypher

**波场 (TRX)**:
- TronScan
- TronGrid

**价格查询**:
- CoinGecko

### 2. 自动切换机制

- 按优先级顺序尝试调用
- 失败自动切换到下一个平台
- 所有平台失败才抛出错误
- 详细的日志记录

### 3. 缓存机制

- 余额缓存 60 秒
- 交易缓存 120 秒
- 代币缓存 120 秒
- Redis 降级策略

### 4. API Key 支持

- 支持配置 API Key
- 留空使用免费 API
- 自动添加到请求头或参数

## 实现细节

### 以太坊服务

**支持的功能**:
- `getEthBalance()` - 获取 ETH 余额
- `getEthTransactions()` - 获取交易记录
- `getEthTokens()` - 获取 ERC-20 代币

**平台实现**:
- `getBalanceFromEtherscan()` - Etherscan API
- `getBalanceFromEthplorer()` - Ethplorer API
- `getBalanceFromBlockscout()` - Blockscout API

### 比特币服务

**支持的功能**:
- `getBtcBalance()` - 获取 BTC 余额
- `getBtcTransactions()` - 获取交易记录

**平台实现**:
- `getBalanceFromBlockchainInfo()` - Blockchain.info API
- `getBalanceFromBlockchair()` - Blockchair API
- `getBalanceFromBlockcypher()` - Blockcypher API

### 波场服务

**支持的功能**:
- `getTrxBalance()` - 获取 TRX 余额
- `getTrxTransactions()` - 获取交易记录（TRX + TRC-20）
- `getTrxTokens()` - 获取 TRC-20 代币

**平台实现**:
- `getBalanceFromTronScan()` - TronScan API
- `getBalanceFromTronGrid()` - TronGrid API

**特殊处理**:
- 同时获取 TRX 和 TRC-20 交易
- 合并并按时间排序
- 支持 USDT-TRC20 查询

### 价格服务

**支持的功能**:
- `getPrices()` - 批量获取价格
- `getEthPrice()` - 获取 ETH 价格
- `getBtcPrice()` - 获取 BTC 价格
- `getTrxPrice()` - 获取 TRX 价格

**平台实现**:
- `getPricesFromCoinGecko()` - CoinGecko API

## 使用流程

### 1. 配置 API 平台

1. 登录后台 `/admin`
2. 进入对应的 API 配置页面
3. 配置平台信息（URL、API Key、优先级）
4. 保存配置

### 2. 系统自动应用

- 配置保存到 MongoDB
- 服务层自动读取配置
- 按优先级依次尝试调用
- 失败自动切换到下一个平台

### 3. 查询地址

- 用户在前端输入地址
- 系统调用对应的服务函数
- 服务函数使用配置的平台
- 返回查询结果

## 错误处理

### 1. 平台失败

```typescript
try {
  const result = await callApi(platform);
  return result;
} catch (error) {
  console.error(`${platform.name} 失败:`, error);
  // 自动尝试下一个平台
}
```

### 2. 所有平台失败

```typescript
throw new Error(
  `所有 ${chainType} API 平台都失败了。最后错误: ${lastError?.message}`
);
```

### 3. 缓存降级

```typescript
// Redis 失败时直接调用 API
if (!redis.isConnected()) {
  return await fetchData();
}
```

## 性能优化

### 1. 缓存策略

- 余额数据缓存 60 秒

### 2. 并发控制

- 使用 Redis 缓存避免重复请求
