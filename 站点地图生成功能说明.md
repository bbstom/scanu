# 站点地图生成功能说明

## 功能概述

站点地图（Sitemap）生成功能允许管理员在后台手动生成包含动态地址页面的 sitemap.xml 文件，显著提升网站的 SEO 效果和搜索引擎收录率。

## 功能特性

### 1. 智能地址选择
- **热门地址**: 按搜索次数排序，优先包含最受欢迎的地址
- **最近搜索**: 按最后搜索时间排序，包含最新查询的地址
- **混合模式**: 同时包含热门和最近搜索，自动去重

### 2. 灵活配置
- **可选包含类型**: 选择包含热门地址、最近搜索或两者
- **URL 数量控制**: 10-500 个动态 URL，可自由调节
- **实时统计**: 显示各链类型的地址数量

### 3. 多种输出方式
- **在线预览**: 直接在页面查看生成的 XML
- **一键下载**: 下载为 sitemap.xml 文件
- **复制功能**: 快速复制到剪贴板

## 使用方法

### 步骤 1: 访问站点地图管理
1. 登录管理后台
2. 点击左侧菜单"站点地图"
3. 查看当前统计信息

### 步骤 2: 配置生成选项
1. **选择包含类型**:
   - ☑️ 包含热门地址
   - ☑️ 包含最近搜索
   - 可以同时选择或单独选择

2. **设置 URL 数量**:
   - 拖动滑块选择 10-500
   - 建议值: 100-200

### 步骤 3: 生成站点地图
1. 点击"生成站点地图"按钮
2. 等待生成完成（通常 1-3 秒）
3. 查看生成结果和 URL 数量

### 步骤 4: 使用生成的文件
**方式一: 下载文件**
1. 点击"下载 sitemap.xml"按钮
2. 将文件上传到网站根目录
3. 确保可以通过 `https://yoursite.com/sitemap.xml` 访问

**方式二: 复制内容**
1. 点击"复制"按钮
2. 手动创建 sitemap.xml 文件
3. 粘贴内容并保存

## 站点地图结构

### 静态页面（固定包含）
```xml
<url>
  <loc>https://yoursite.com/</loc>
  <lastmod>2024-03-01T12:00:00.000Z</lastmod>
  <changefreq>daily</changefreq>
  <priority>1.0</priority>
</url>
<url>
  <loc>https://yoursite.com/guide</loc>
  <lastmod>2024-03-01T12:00:00.000Z</lastmod>
  <changefreq>weekly</changefreq>
  <priority>0.8</priority>
</url>
<url>
  <loc>https://yoursite.com/faq</loc>
  <lastmod>2024-03-01T12:00:00.000Z</lastmod>
  <changefreq>weekly</changefreq>
  <priority>0.7</priority>
</url>
```

### 动态地址页面（根据配置生成）
```xml
<url>
  <loc>https://yoursite.com/eth/0x1234...</loc>
  <lastmod>2024-03-01T10:30:00.000Z</lastmod>
  <changefreq>daily</changefreq>
  <priority>0.6</priority>
</url>
<url>
  <loc>https://yoursite.com/btc/1A2B3C...</loc>
  <lastmod>2024-03-01T09:15:00.000Z</lastmod>
  <changefreq>daily</changefreq>
  <priority>0.6</priority>
</url>
<url>
  <loc>https://yoursite.com/trx/TMfxcr...</loc>
  <lastmod>2024-03-01T08:45:00.000Z</lastmod>
  <changefreq>daily</changefreq>
  <priority>0.6</priority>
</url>
```

## 提交到搜索引擎

### Google Search Console
1. 登录 [Google Search Console](https://search.google.com/search-console)
2. 选择您的网站
3. 点击左侧"站点地图"
4. 输入 `sitemap.xml`
5. 点击"提交"

### Bing Webmaster Tools
1. 登录 [Bing Webmaster Tools](https://www.bing.com/webmasters)
2. 选择您的网站
3. 点击"站点地图"
4. 输入 `https://yoursite.com/sitemap.xml`
5. 点击"提交"

### 百度站长平台
1. 登录 [百度站长平台](https://ziyuan.baidu.com)
2. 选择您的网站
3. 点击"数据引入" > "链接提交"
4. 选择"sitemap"
5. 输入 sitemap URL 并提交

## 更新频率建议

### 高流量网站
- **频率**: 每天更新
- **原因**: 新地址查询频繁，需要及时收录

### 中等流量网站
- **频率**: 每周更新
- **原因**: 平衡 SEO 效果和维护成本

### 低流量网站
- **频率**: 每月更新
- **原因**: 内容变化较少

## 最佳实践

### 1. URL 数量控制
- **推荐**: 100-200 个动态 URL
- **原因**: 
  - 太少：无法充分展示网站内容
  - 太多：可能被搜索引擎视为低质量

### 2. 内容选择策略
- **优先热门地址**: 搜索次数多，用户关注度高
- **包含最近搜索**: 保持内容新鲜度
- **混合模式**: 平衡热度和新鲜度

### 3. 定期更新
- 设置提醒，定期生成新的站点地图
- 监控搜索引擎收录情况
- 根据效果调整配置

### 4. 验证有效性
使用工具验证站点地图：
- [XML Sitemap Validator](https://www.xml-sitemaps.com/validate-xml-sitemap.html)
- Google Search Console 的站点地图报告

## 技术实现

### API 端点
- **GET** `/api/admin/sitemap` - 获取统计数据
- **POST** `/api/admin/sitemap` - 生成站点地图

### 数据来源
从 `SearchHistory` 集合读取：
- 地址 (address)
- 链类型 (chainType)
- 搜索次数 (searchCount)
- 最后搜索时间 (lastSearchAt)

### 生成逻辑
```typescript
// 热门地址
SearchHistory.find()
  .sort({ searchCount: -1 })
  .limit(maxUrls)

// 最近搜索
SearchHistory.find()
  .sort({ lastSearchAt: -1 })
  .limit(maxUrls)

// 混合模式（自动去重）
const popular = await SearchHistory.find().sort({ searchCount: -1 }).limit(50);
const recent = await SearchHistory.find().sort({ lastSearchAt: -1 }).limit(50);
const combined = [...popular, ...recent];
const unique = removeDuplicates(combined);
```

## 故障排查

### 问题 1: 生成失败
**可能原因**:
- 数据库连接问题
- 权限不足
- 数据为空

**解决方法**:
1. 检查 MongoDB 连接
2. 确认已登录管理后台
3. 确保有搜索历史数据

### 问题 2: 下载的文件无法使用
**可能原因**:
- 文件编码问题
- XML 格式错误

**解决方法**:
1. 使用 UTF-8 编码保存
2. 使用 XML 验证工具检查
3. 重新生成站点地图

### 问题 3: 搜索引擎无法访问
**可能原因**:
- 文件路径错误
- 服务器配置问题
- robots.txt 阻止

**解决方法**:
1. 确保文件在网站根目录
2. 检查文件权限（644）
3. 检查 robots.txt 配置
