# 构建优化完成总结

## 📋 问题描述

用户在宝塔面板部署时遇到构建错误：
```
Next.js build worker exited with code: null and signal: SIGBUS
```

**根本原因：** 配置要求 4GB 内存，但服务器内存不足。

---

## ✅ 已完成的优化

### 1. 配置文件优化

#### package.json
```diff
- "dev": "node --max-old-space-size=4096 node_modules/next/dist/bin/next dev"
- "build": "node --max-old-space-size=4096 node_modules/next/dist/bin/next build"
+ "dev": "next dev"
+ "build": "next build"
+ "build:production": "NODE_OPTIONS='--max-old-space-size=512' next build"
```

#### .npmrc
```diff
- node-options=--max-old-space-size=4096
+ # 生产环境使用较小的内存限制
+ # node-options=--max-old-space-size=512
```

#### next.config.mjs
```javascript
// 新增优化配置
{
  swcMinify: true,
  experimental: {
    workerThreads: false,
    cpus: 1,
  },
  webpack: (config) => {
    config.optimization = {
      ...config.optimization,
      minimize: true,
      splitChunks: {
        chunks: 'all',
        cacheGroups: {
          default: false,
          vendors: false,
        },
      },
    };
    return config;
  }
}
```

## 🔧 技术细节

### 1. 内存优化原理

**减少并行编译：**
```javascript
experimental: {
  workerThreads: false,  // 禁用 worker 线程
  cpus: 1,               // 使用单核编译
}
```

**优化代码分割：**
```javascript
splitChunks: {
  chunks: 'all',
  cacheGroups: {
    default: false,      // 禁用默认缓存组
    vendors: false,      // 禁用 vendors 缓存组
  },
}
```

**使用 SWC 压缩：**
```javascript
swcMinify: true,  // 使用 Rust 编写的 SWC，比 Terser 更快更省内存
```

### 2. Swap 配置

**为什么需要 Swap？**
- 构建时内存峰值可能超过物理内存
- Swap 作为虚拟内存的扩展
- 防止 OOM (Out of Memory) 错误

**Swap 大小建议：**
- 物理内存 < 1GB：创建 2GB Swap
- 物理内存 1-2GB：创建 1GB Swap
- 物理内存 > 2GB：可选

**Swappiness 设置：**
```bash
vm.swappiness=10  # 优先使用物理内存，只在必要时使用 Swap
```

### 3. 构建流程优化

**优化前：**
```
安装依赖 → 多线程编译 → 并行打包 → 压缩（Terser）
内存峰值：~4GB
```

**优化后：**
```
安装依赖 → 单线程编译 → 优化打包 → 压缩（SWC）
内存峰值：~512MB
```

## 🎯 适用场景

### ✅ 适合使用优化配置

- 小内存服务器（< 2GB）
- 宝塔面板等共享主机
- VPS 或云服务器（基础配置）
- 开发环境
- 测试环境

### ⚠️ 可选使用优化配置

- 中等内存服务器（2-4GB）
- 专用服务器
- 生产环境（如果内存充足）

### ❌ 不建议使用优化配置

- 大内存服务器（> 8GB）
- 需要快速构建的 CI/CD 环境
- 频繁构建的开发环境

**注意：** 对于大内存服务器，可以恢复原配置以获得更快的构建速度。
