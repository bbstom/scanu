# 简易部署指南

## 最简单的部署方式

### 前置要求
- Node.js 18+
- MongoDB（本地或远程）
- Redis（可选，但推荐）

---

## 方式 1: 直接运行（最简单）

### 步骤 1: 配置环境变量

复制 `.env.example` 为 `.env`，然后修改：

```bash
# 复制配置文件
cp .env.example .env

# 编辑配置文件
nano .env
```

修改以下内容：

```env
# MongoDB 连接（必需）
MONGODB_URI=mongodb://localhost:27017/crypto-wallet-explorer

# Redis（可选）
REDIS_URL=redis://localhost:6379
REDIS_ENABLED=true

# 管理员设置密钥（重要！）
ADMIN_SETUP_KEY=your-secure-random-key

# 应用 URL
NEXT_PUBLIC_APP_URL=http://your-server-ip:3000
NEXT_PUBLIC_BASE_URL=http://your-server-ip:3000

# 环境
NODE_ENV=production
```

### 步骤 2: 安装依赖

```bash
npm install
```

### 步骤 3: 构建项目

```bash
npm run build
```

### 步骤 4: 启动应用

```bash
npm start
```

应用将在 `http://localhost:3000` 运行。

### 保持后台运行

如果你关闭终端，应用会停止。要保持后台运行，可以使用 `nohup`：

```bash
nohup npm start > app.log 2>&1 &
```

查看日志：
```bash
tail -f app.log
```

停止应用：
```bash
# 查找进程
ps aux | grep node

# 杀死进程
kill <进程ID>
```

## 方式 2: 使用 Docker（推荐）

### 为什么推荐 Docker？
- ✅ 一键启动所有服务（应用 + MongoDB + Redis）
- ✅ 自动重启
- ✅ 环境隔离
- ✅ 易于管理

### 步骤 1: 安装 Docker

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo apt install docker-compose
```

### 步骤 2: 修改配置

编辑 `docker-compose.yml`，修改以下内容：

```yaml
environment:
  # 修改 MongoDB 密码
  - MONGODB_URI=mongodb://admin:你的密码@mongodb:27017/crypto-wallet-explorer?authSource=admin
  # 修改管理员密钥
  - ADMIN_SETUP_KEY=你的安全密钥
  # 修改域名
  - NEXT_PUBLIC_APP_URL=http://你的服务器IP:3000
  - NEXT_PUBLIC_BASE_URL=http://你的服务器IP:3000
```

同时修改 MongoDB 密码：

```yaml
mongodb:
  environment:
    MONGO_INITDB_ROOT_PASSWORD: 你的密码
```

### 步骤 3: 启动所有服务

```bash
# 启动（首次会自动构建）
docker-compose up -d

# 查看日志
docker-compose logs -f app

# 查看状态
docker-compose ps
```

### 常用命令

```bash
# 重启应用
docker-compose restart app

# 停止所有服务
docker-compose down

# 更新代码后重新构建
docker-compose up -d --build

# 查看日志
docker-compose logs -f app
```

## 方式 3: 使用 PM2（可选，用于进程管理）

### 为什么使用 PM2？
- ✅ 自动重启（崩溃时）
- ✅ 负载均衡（多进程）
- ✅ 日志管理
- ✅ 开机自启

### 步骤 1: 安装 PM2

```bash
npm install -g pm2
```

### 步骤 2: 配置环境变量

同方式 1，配置 `.env` 文件

### 步骤 3: 构建项目

```bash
npm install
npm run build
```

### 步骤 4: 使用 PM2 启动

```bash
# 启动应用
pm2 start npm --name "crypto-app" -- start

# 查看状态
pm2 status

# 查看日志
pm2 logs crypto-app

# 重启
pm2 restart crypto-app

# 停止
pm2 stop crypto-app

# 删除
pm2 delete crypto-app
```

### 设置开机自启

```bash
# 保存当前进程列表
pm2 save

# 设置开机自启
pm2 startup

# 按照提示执行命令（通常是一条 sudo 命令）
```

## 部署后配置

### 1. 访问应用

打开浏览器访问：`http://你的服务器IP:3000`

### 2. 初始化管理员

访问：`http://你的服务器IP:3000/admin/setup`

1. 输入你在 `.env` 中设置的 `ADMIN_SETUP_KEY`
2. 创建管理员账号
3. 登录管理后台

### 3. 配置 API 密钥（可选）

在管理后台配置：
- Etherscan API Key（以太坊）
- TronGrid API Key（波场）
- CoinGecko API（价格）

这些都是免费的，可以提高查询速度和稳定性。

## 使用 Nginx 反向代理（可选）

如果你想：
- 使用域名访问
- 启用 HTTPS
- 隐藏端口号

### 步骤 1: 安装 Nginx

```bash
sudo apt install nginx
```

### 步骤 2: 创建配置文件

```bash
sudo nano /etc/nginx/sites-available/crypto-app
```

粘贴以下内容：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 步骤 3: 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/crypto-app /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 步骤 4: 配置 HTTPS（使用 Let's Encrypt）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期（Certbot 会自动设置）
sudo certbot renew --dry-run
```

## 对比三种方式

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **直接运行** | 最简单，快速启动 | 需要手动管理，关闭终端会停止 | 测试、临时部署 |
| **Docker** | 一键启动所有服务，易于管理 | 需要学习 Docker | 推荐用于生产环境 |
| **PM2** | 自动重启，进程管理 | 需要单独安装 MongoDB/Redis | 传统服务器部署 |

## 常见问题

### Q: 必须使用 PM2 吗？
A: 不是必须的。`npm start` 就可以运行，PM2 只是提供更好的进程管理。

### Q: 必须使用 Docker 吗？
A: 不是必须的。Docker 只是让部署更简单，你可以直接用 `npm start`。

### Q: Redis 是必需的吗？
A: 不是必需的。没有 Redis，应用会直接调用 API，只是会慢一些。

### Q: 如何更新应用？
A: 
```bash
# 拉取最新代码
git pull

# 重新构建
npm run build

# 重启应用
# 如果用 npm start: 杀死进程后重新启动
# 如果用 PM2: pm2 restart crypto-app
# 如果用 Docker: docker-compose restart app
```

### Q: 如何查看日志？
A:
```bash
# npm start: 查看 app.log
tail -f app.log

# PM2: 
pm2 logs crypto-app

# Docker:
docker-compose logs -f app
```
