# Next.js 端口配置说明

## 修改运行端口的方法

### 方法 1：使用环境变量（推荐）

在 `.env` 文件中设置：

```env
PORT=3000
HOSTNAME=0.0.0.0
```

然后启动：
```bash
npm start
```

### 方法 2：启动时指定端口

```bash
# 直接启动
npm start -- -p 8080

# 或使用 next 命令
npx next start -p 8080

# 指定主机和端口
npx next start -H 0.0.0.0 -p 8080
```

### 方法 3：使用 PM2 指定端口

```bash
# 方法 A：使用环境变量
PORT=8080 pm2 start npm --name crypto-app -- start

# 方法 B：使用 PM2 配置文件（见下方）
pm2 start ecosystem.config.js
```

### 方法 4：修改 package.json

在 `package.json` 中修改 start 脚本：

```json
{
  "scripts": {
    "start": "next start -p 8080"
  }
}
```

## 常用端口

| 端口 | 用途 |
|------|------|
| 3000 | Next.js 默认端口 |
| 8080 | 常用的 Web 应用端口 |
| 3001 | 备用端口 |
| 80 | HTTP（需要 root 权限） |
| 443 | HTTPS（需要 root 权限） |

## PM2 配置文件

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'crypto-app',
    script: 'npm',
    args: 'start',
    env: {
      NODE_ENV: 'production',
      PORT: 3000,
      HOSTNAME: '0.0.0.0'
    },
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '500M',
  }]
};
```

使用：
```bash
pm2 start ecosystem.config.js
pm2 save
```

## 宝塔面板配置

### 1. 修改应用端口

在 `.env` 文件中设置：
```env
PORT=3000
```

### 2. 配置反向代理

在宝塔面板：
1. 网站 → 设置 → 反向代理
2. 目标URL：`http://127.0.0.1:3000`（改为你的端口）
3. 发送域名：`$host`

### 3. 如果使用多个端口

可以运行多个实例：

```bash
# 实例 1 - 端口 3000
PORT=3000 pm2 start npm --name crypto-app-1 -- start

# 实例 2 - 端口 3001
PORT=3001 pm2 start npm --name crypto-app-2 -- start
```

## 检查端口占用

### 查看端口是否被占用

```bash
# 方法 1
netstat -tlnp | grep 3000

# 方法 2
lsof -i :3000

# 方法 3
ss -tlnp | grep 3000
```

### 杀死占用端口的进程

```bash
# 查找进程 ID
lsof -ti :3000

# 杀死进程
kill -9 $(lsof -ti :3000)

# 或使用 PM2
pm2 stop crypto-app
pm2 delete crypto-app
```

## 防火墙配置

如果需要外网访问，确保端口已开放：

### Ubuntu/Debian (UFW)

```bash
# 开放端口
sudo ufw allow 3000

# 查看状态
sudo ufw status
```

### CentOS/RHEL (firewalld)

```bash
# 开放端口
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload

# 查看状态
sudo firewall-cmd --list-ports
```

## 常见问题

### Q: 修改端口后无法访问？

A: 检查：
1. 防火墙是否开放端口
2. 反向代理配置是否正确
3. 应用是否正常启动：`pm2 logs crypto-app`

### Q: 端口被占用怎么办？

A: 
```bash
# 查看占用进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=3001 npm start
```

### Q: 如何同时运行多个端口？

A: 使用 PM2 运行多个实例：
```bash
PORT=3000 pm2 start npm --name app-3000 -- start
PORT=3001 pm2 start npm --name app-3001 -- start
```

### Q: 修改端口后需要重新构建吗？

A: 不需要，端口是运行时配置，不影响构建。
