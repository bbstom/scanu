# 网站品牌配置说明

## 概述

网站配置功能已增强，现在支持完整的品牌自定义，包括 Logo 图片、Favicon 图标和浏览器标签页标题。

## 新增配置项

### 1. 浏览器标签页标题 (SITE_TITLE)
- **说明**: 显示在浏览器标签页上的完整标题
- **默认值**: "加密钱包查询工具 - 支持ETH/BTC/TRX多链查询"
- **使用场景**: 浏览器标签页、书签标题
- **示例**: "我的钱包查询 - 专业的区块链查询工具"

### 2. 网站 Logo 图片 URL (SITE_LOGO_URL)
- **说明**: 网站 Logo 图片的 URL 地址
- **默认值**: 空（显示文字 Logo）
- **支持格式**: 
  - 外部链接: `https://example.com/logo.png`
  - 本地路径: `/logo.png` (需要将图片放在 `public` 目录)
- **推荐尺寸**: 高度 40-60px，宽度自适应
- **使用场景**: 页面左上角的网站标识
- **备注**: 如果图片加载失败，会自动回退显示文字 Logo

### 3. 网站 Favicon 图标 URL (SITE_FAVICON_URL)
- **说明**: 浏览器标签页图标的 URL 地址
- **默认值**: 空（使用浏览器默认图标）
- **支持格式**: 
  - 外部链接: `https://example.com/favicon.ico`
  - 本地路径: `/favicon.ico` (需要将图片放在 `public` 目录)
- **推荐格式**: ICO、PNG、SVG
- **推荐尺寸**: 32x32 或 64x64 像素
- **使用场景**: 浏览器标签页图标、书签图标、移动设备主屏幕图标

### 4. 网站 Logo 文字 (SITE_LOGO_TEXT)
- **说明**: 当 Logo 图片为空或加载失败时显示的文字
- **默认值**: "加密钱包查询"
- **使用场景**: Logo 图片的备用方案

## 配置方法

### 方式一：后台管理界面（推荐）

1. 登录管理后台: `/admin`
2. 进入"网站配置"页面
3. 填写各项配置
4. 点击"保存配置"

### 方式二：直接修改数据库

```javascript
// MongoDB 中的 settings 集合
{
  key: "SITE_TITLE",
  value: "我的钱包查询工具",
  description: "浏览器标签页标题"
}
```

## 使用本地图片

### 步骤 1: 准备图片文件

将图片文件放在项目的 `public` 目录下：

```
public/
  ├── logo.png          # 网站 Logo
  ├── favicon.ico       # Favicon 图标
  └── favicon-32x32.png # 备用 Favicon
```

### 步骤 2: 配置路径

在后台管理界面中填写：

- **Logo 图片 URL**: `/logo.png`
- **Favicon 图标 URL**: `/favicon.ico`

### 步骤 3: 重启应用

```bash
npm run build
npm start
```

## 使用外部图片

### 示例配置

- **Logo 图片 URL**: `https://cdn.example.com/images/logo.png`
- **Favicon 图标 URL**: `https://cdn.example.com/images/favicon.ico`

### 注意事项

1. 确保图片 URL 可公开访问
2. 建议使用 HTTPS 协议
3. 注意跨域问题（CORS）
4. 建议使用 CDN 加速

## 图片规格建议

### Logo 图片
- **格式**: PNG（支持透明背景）、SVG（矢量图）
- **尺寸**: 高度 40-60px，宽度根据实际 Logo 比例
- **文件大小**: < 100KB
- **背景**: 透明背景更佳

### Favicon 图标
- **格式**: ICO（兼容性最好）、PNG、SVG
- **尺寸**: 
  - 标准: 32x32px
  - 高清: 64x64px
  - 多尺寸 ICO: 16x16, 32x32, 48x48
- **文件大小**: < 50KB

## 配置示例

### 示例 1: 使用本地图片

```
SITE_TITLE = "CryptoWallet Explorer - 专业的区块链查询工具"
SITE_LOGO_URL = "/logo.png"
SITE_LOGO_TEXT = "CryptoWallet"
SITE_FAVICON_URL = "/favicon.ico"
```

### 示例 2: 使用外部图片

```
SITE_TITLE = "我的钱包查询 - 支持多链查询"
SITE_LOGO_URL = "https://cdn.mysite.com/logo.png"
SITE_LOGO_TEXT = "我的钱包"
SITE_FAVICON_URL = "https://cdn.mysite.com/favicon.ico"
```

### 示例 3: 仅使用文字 Logo

```
SITE_TITLE = "加密钱包查询工具"
SITE_LOGO_URL = ""  # 留空
SITE_LOGO_TEXT = "🔍 钱包查询"
SITE_FAVICON_URL = ""  # 留空，使用默认
```

## 预览功能

在后台管理界面中：

- **Logo 图片**: 输入 URL 后会实时显示预览
- **Favicon 图标**: 输入 URL 后会显示小图标预览
- **加载失败提示**: 如果图片 URL 无效，会显示错误提示

## 故障排查

### Logo 图片不显示

1. 检查图片 URL 是否正确
2. 检查图片是否可公开访问
3. 检查浏览器控制台是否有错误
4. 尝试清除浏览器缓存
5. 确认图片格式是否支持（PNG、JPG、SVG）

### Favicon 不显示

1. 检查图片 URL 是否正确
2. 清除浏览器缓存（Favicon 缓存时间较长）
3. 尝试使用 ICO 格式
4. 确认图片尺寸是否合适（推荐 32x32）
5. 重启浏览器

### 配置不生效

1. 确认已点击"保存配置"按钮
2. 检查是否有保存成功的提示
3. 刷新页面（Ctrl+F5 强制刷新）
4. 检查 MongoDB 数据库中的配置是否已更新
5. 重启 Next.js 应用

## 技术实现

### 配置读取

```typescript
// 从数据库读取配置
const logoUrl = await getSetting("SITE_LOGO_URL", "");
const faviconUrl = await getSetting("SITE_FAVICON_URL", "");
const siteTitle = await getSetting("SITE_TITLE", "默认标题");
```

### Logo 显示逻辑

```tsx
{logoUrl ? (
  <img src={logoUrl} alt={logoText} className="h-10" />
) : (
  <span>{logoText}</span>
)}
```

### Favicon 配置

```typescript
metadata.icons = {
  icon: faviconUrl,
  shortcut: faviconUrl,
  apple: faviconUrl,
};
```

## 最佳实践

1. **使用 CDN**: 将图片托管在 CDN 上，提高加载速度
2. **优化图片**: 压缩图片文件大小，提升性能
3. **提供备用方案**: 设置好文字 Logo，防止图片加载失败
4. **统一风格**: Logo 和 Favicon 保持视觉一致性
5. **测试兼容性**: 在不同浏览器和设备上测试显示效果
6. **定期检查**: 定期检查外部图片链接是否有效
