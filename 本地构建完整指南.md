# 本地构建完整指南

## 为什么需要本地构建？

你的服务器内存太小（可能 < 512MB），无法完成构建过程。但是：
- ✅ 本地电脑有足够的内存
- ✅ 构建后的文件很小（~50MB）
- ✅ 服务器运行只需要 200MB 内存

所以最佳方案是：**在本地构建，上传到服务器运行**

---

## 方案 A：使用自动化脚本（推荐）

### 在 Windows 本地电脑：

1. 打开项目目录：`E:\vscodefile\usdtcheck`

2. 双击运行：`本地构建.bat`

3. 等待构建完成，会生成 `build.zip` 文件

### 上传到服务器：

1. 在宝塔面板，点击 **文件**
2. 进入 `/www/wwwroot/scanu.cc/scanu`
3. 上传 `build.zip`
4. 右键点击 `build.zip`，选择 **解压**

### 在服务器启动：

```bash
cd /www/wwwroot/scanu.cc/scanu
npm ci --production
pm2 start npm --name crypto-app -- start
pm2 save
```

## 方案 B：手动操作

### 步骤 1：在本地构建

打开 PowerShell 或 CMD：

```bash
# 进入项目目录
cd E:\vscodefile\usdtcheck

# 安装依赖（如果还没安装）
npm install

# 清理旧构建
Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue

# 构建
npm run build
```

### 步骤 2：打包文件

#### 方法 A：使用 PowerShell（推荐）

```powershell
# 创建压缩包
Compress-Archive -Path .next, package.json, package-lock.json, public, next.config.mjs -DestinationPath build.zip -Force
```

#### 方法 B：使用 7-Zip 或 WinRAR

1. 选择以下文件/文件夹：
   - `.next` 文件夹（必需）
   - `package.json`（必需）
   - `package-lock.json`（必需）
   - `public` 文件夹（如果有）
   - `next.config.mjs`（必需）

2. 右键 → 添加到压缩文件 → 命名为 `build.zip`

### 步骤 3：上传到服务器

#### 方法 A：使用宝塔面板（推荐）

1. 登录宝塔面板
2. 点击左侧 **文件**
3. 导航到 `/www/wwwroot/scanu.cc/scanu`
4. 点击 **上传**
5. 选择 `build.zip`
6. 等待上传完成
7. 右键点击 `build.zip` → **解压**

#### 方法 B：使用 SCP（命令行）

```bash
# 在本地 PowerShell 中
scp build.zip root@your-server-ip:/www/wwwroot/scanu.cc/scanu/
```

#### 方法 C：使用 FTP 工具

使用 FileZilla 或其他 FTP 工具上传 `build.zip`

### 步骤 4：在服务器上部署

```bash
# SSH 连接到服务器
ssh root@your-server-ip

# 进入项目目录
cd /www/wwwroot/scanu.cc/scanu

# 解压（如果还没解压）
unzip -o build.zip

# 检查 .next 目录
ls -lh .next

# 安装生产依赖
npm ci --production

# 启动应用
pm2 start npm --name crypto-app -- start
pm2 save
pm2 startup
```

## 常见问题

### Q1: 本地构建失败？

**检查 Node.js 版本：**
```bash
node -v
# 需要 v18 或更高
```

**清理缓存重试：**
```bash
npm cache clean --force
Remove-Item -Recurse -Force node_modules, .next
npm install
npm run build
```

### Q2: 上传很慢？

**压缩文件太大：**
- 只打包必需文件（.next, package.json, package-lock.json）
- 不要包含 node_modules
- 使用更高的压缩率

**网络问题：**
- 使用宝塔面板上传（支持断点续传）
- 或使用 FTP 工具

### Q3: 解压失败？

**在服务器上手动解压：**
```bash
cd /www/wwwroot/scanu.cc/scanu
unzip -o build.zip
```

**或使用 tar（如果打包为 .tar.gz）：**
```bash
tar -xzf build.tar.gz
```

### Q4: 启动失败？

**检查依赖：**
```bash
npm ci --production
```

**检查环境变量：**
```bash
cat .env
# 确保配置正确
```

**查看日志：**
```bash
pm2 logs crypto-app
```

### Q5: 如何更新代码？

每次更新代码后：

1. 在本地重新构建：
```bash
npm run build
```

2. 重新打包上传

3. 在服务器重启：
```bash
pm2 restart crypto-app
```

## 文件清单

构建完成后，确保包含以下文件：

```
build.zip
├── .next/                    # 构建输出（必需）
│   ├── cache/
│   ├── server/
│   ├── static/
│   └── build-manifest.json
├── package.json              # 依赖配置（必需）
├── package-lock.json         # 锁定版本（必需）
├── next.config.mjs           # Next.js 配置（必需）
└── public/                   # 静态资源（如果有）
```

**不要包含：**
- ❌ `node_modules/` - 太大，服务器会重新安装
- ❌ `.git/` - 不需要
- ❌ `src/` - 已编译到 .next
- ❌ 文档文件 - 不需要
