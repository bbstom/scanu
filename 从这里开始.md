## 🔧 手动修复（如果脚本失败）

### 方法 1：清理后重建

```bash
cd /www/wwwroot/scanu.cc/scanu

# 清理
rm -rf .next node_modules/.cache .swc
npm cache clean --force

# 重新构建
npm run build
```

### 方法 2：禁用 SWC 编译器

如果是 SWC 兼容性问题：

```bash
cd /www/wwwroot/scanu.cc/scanu
rm -rf .next

# 使用 Babel 代替 SWC
NEXT_DISABLE_SWC=1 npm run build
```

## 📋 详细步骤（如果一键部署失败）

### 步骤 1：检查环境

```bash
# 检查 Node.js 版本（需要 v18+）
node -v

# 检查内存
free -h

# 检查磁盘空间
df -h
```

### 步骤 2：创建 Swap（如果内存 < 1GB）

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 步骤 3：配置环境变量

```bash
# 复制环境变量文件
cp .env.example .env

# 编辑 .env（在宝塔面板文件管理中编辑）
# 或使用命令行：nano .env
```

必填项：
```env
MONGODB_URI=mongodb://localhost:27017/crypto-explorer
ADMIN_PASSWORD=your-secure-password
NEXT_PUBLIC_SITE_URL=https://scanu.cc
```

### 步骤 4：安装依赖

```bash
npm install
```

### 步骤 5：构建项目

```bash
# 方法 A：使用 512MB 内存（推荐）
NODE_OPTIONS='--max-old-space-size=512' npm run build

# 方法 B：如果方法 A 失败，使用 256MB
NODE_OPTIONS='--max-old-space-size=256' npm run build

# 方法 C：使用默认内存
npm run build
```

### 步骤 6：启动应用

```bash
# 使用 PM2（推荐）
pm2 start npm --name crypto-app -- start
pm2 save
pm2 startup

# 或直接启动
npm start
```

## 🌐 配置域名访问

### 在宝塔面板中：

1. **添加网站**
   - 点击 "网站" → "添加站点"
   - 域名：`scanu.cc`
   - 根目录：`/www/wwwroot/scanu.cc`
   - PHP版本：纯静态

2. **配置反向代理**
   - 点击站点 "设置" → "反向代理"
   - 添加反向代理：
     - 代理名称：`crypto-app`
     - 目标URL：`http://127.0.0.1:3000`
     - 发送域名：`$host`

3. **申请 SSL 证书**
   - 点击 "SSL" → "Let's Encrypt"
   - 勾选域名 → 申请
   - 开启 "强制HTTPS"

## ❓ 常见问题

### Q: 构建时出现 SIGBUS 错误？
**A:** 内存不足，执行以下命令创建 Swap：
```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Q: 构建需要多长时间？
**A:** 通常 3-10 分钟，取决于服务器性能。

### Q: 如何查看构建进度？
**A:** 构建过程会实时显示进度，耐心等待即可。

### Q: 构建成功后如何验证？
**A:** 
```bash
# 检查 .next 目录
ls -lh .next

# 测试访问
curl http://localhost:3000
```

### Q: 如何查看应用日志？
**A:**
```bash
pm2 logs crypto-app
```

### Q: 如何重启应用？
**A:**
```bash
pm2 restart crypto-app
```

### Q: 端口 3000 被占用？
**A:**
```bash
# 查看占用进程
netstat -tlnp | grep 3000

# 杀死进程
kill -9 <PID>
```
