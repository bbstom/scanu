# 生产环境部署指南

## 部署前检查清单

### 1. 环境要求
- ✅ Node.js 18+ 
- ✅ MongoDB 4.4+
- ✅ Redis 6+ (可选，但强烈推荐)
- ✅ 至少 2GB RAM
- ✅ 至少 10GB 磁盘空间

### 2. 必需配置
- ✅ MongoDB 数据库已安装并运行
- ✅ 生产环境域名已准备
- ✅ SSL 证书已准备（推荐使用 Let's Encrypt）
- ✅ 防火墙规则已配置

## 部署方式选择

### 方式 1: Docker 部署（推荐）
适合快速部署和容器化环境

### 方式 2: PM2 部署
适合传统服务器环境

### 方式 3: Vercel/Netlify 部署
适合无服务器环境（需要外部 MongoDB）

## 方式 1: Docker 部署（推荐）

### 步骤 1: 准备环境变量

创建生产环境配置文件 `.env.production`:

```bash
# MongoDB 配置（必需）
MONGODB_URI=mongodb://用户名:密码@mongodb服务器:27017/crypto-wallet-explorer?authSource=admin

# Redis 配置（推荐）
REDIS_URL=redis://redis服务器:6379
REDIS_ENABLED=true

# 管理员设置密钥（重要！）
ADMIN_SETUP_KEY=your-very-secure-random-key-here-change-this

# 应用 URL
NEXT_PUBLIC_APP_URL=https://your-domain.com
NEXT_PUBLIC_BASE_URL=https://your-domain.com

# 环境
NODE_ENV=production
```

### 步骤 2: 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  # MongoDB 数据库
  mongodb:
    image: mongo:6
    container_name: crypto-mongodb
    restart: always
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: your-secure-password
      MONGO_INITDB_DATABASE: crypto-wallet-explorer
    volumes:
      - mongodb_data:/data/db
    ports:
      - "27017:27017"
    networks:
      - crypto-network

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: crypto-redis
    restart: always
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"
    networks:
      - crypto-network

  # Next.js 应用
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: crypto-app
    restart: always
    ports:
      - "3000:3000"
    environment:
      - MONGODB_URI=mongodb://admin:your-secure-password@mongodb:27017/crypto-wallet-explorer?authSource=admin
      - REDIS_URL=redis://redis:6379
      - REDIS_ENABLED=true
      - ADMIN_SETUP_KEY=your-very-secure-random-key-here
      - NEXT_PUBLIC_APP_URL=https://your-domain.com
      - NEXT_PUBLIC_BASE_URL=https://your-domain.com
      - NODE_ENV=production
    depends_on:
      - mongodb
      - redis
    networks:
      - crypto-network

  # Nginx 反向代理（可选）
  nginx:
    image: nginx:alpine
    container_name: crypto-nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - app
    networks:
      - crypto-network

volumes:
  mongodb_data:
  redis_data:

networks:
  crypto-network:
    driver: bridge
```

### 步骤 3: 创建 Nginx 配置

创建 `nginx.conf`:

```nginx
events {
    worker_connections 1024;
}

http {
    upstream app {
        server app:3000;
    }

    # HTTP 重定向到 HTTPS
    server {
        listen 80;
        server_name your-domain.com;
        return 301 https://$server_name$request_uri;
    }

    # HTTPS 配置
    server {
        listen 443 ssl http2;
        server_name your-domain.com;

        # SSL 证书
        ssl_certificate /etc/nginx/ssl/fullchain.pem;
        ssl_certificate_key /etc/nginx/ssl/privkey.pem;

        # SSL 配置
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        ssl_prefer_server_ciphers on;

        # 安全头
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Referrer-Policy "no-referrer-when-downgrade" always;

        # 代理配置
        location / {
            proxy_pass http://app;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_cache_bypass $http_upgrade;
        }

        # 静态文件缓存
        location /_next/static {
            proxy_pass http://app;
            proxy_cache_valid 200 60m;
            add_header Cache-Control "public, max-age=3600, immutable";
        }

        # 图片缓存
        location ~* \.(jpg|jpeg|png|gif|ico|svg)$ {
            proxy_pass http://app;
            expires 30d;
            add_header Cache-Control "public, max-age=2592000";
        }
    }
}
```

### 步骤 4: 部署

```bash
# 1. 构建并启动所有服务
docker-compose up -d --build

# 2. 查看日志
docker-compose logs -f app

# 3. 检查服务状态
docker-compose ps

# 4. 停止服务
docker-compose down

# 5. 重启服务
docker-compose restart app
```

## 方式 2: PM2 部署

### 步骤 1: 安装依赖

```bash
# 安装 Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2
sudo npm install -g pm2

# 安装项目依赖
npm ci --production
```

### 步骤 2: 配置环境变量

创建 `.env.production`:

```bash
MONGODB_URI=mongodb://用户名:密码@localhost:27017/crypto-wallet-explorer?authSource=admin
REDIS_URL=redis://localhost:6379
REDIS_ENABLED=true
ADMIN_SETUP_KEY=your-very-secure-random-key-here
NEXT_PUBLIC_APP_URL=https://your-domain.com
NEXT_PUBLIC_BASE_URL=https://your-domain.com
NODE_ENV=production
```

### 步骤 3: 构建项目

```bash
npm run build
```

### 步骤 4: 创建 PM2 配置

创建 `ecosystem.config.js`:

```javascript
module.exports = {
  apps: [{
    name: 'crypto-wallet-explorer',
    script: 'node_modules/next/dist/bin/next',
    args: 'start',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    env_file: '.env.production',
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
    merge_logs: true,
    max_memory_restart: '1G',
    autorestart: true,
    watch: false
  }]
};
```

### 步骤 5: 启动应用

```bash
# 创建日志目录
mkdir -p logs

# 启动应用
pm2 start ecosystem.config.js

# 查看状态
pm2 status

# 查看日志
pm2 logs crypto-wallet-explorer

# 设置开机自启
pm2 startup
pm2 save

# 重启应用
pm2 restart crypto-wallet-explorer

# 停止应用
pm2 stop crypto-wallet-explorer
```

## 方式 3: Vercel 部署

### 步骤 1: 准备外部数据库

确保你有：
- MongoDB Atlas 账号和连接字符串
- Redis Cloud 账号和连接字符串（可选）

### 步骤 2: 配置 Vercel

1. 在 Vercel 中导入项目
2. 配置环境变量：

```
MONGODB_URI=mongodb+srv://用户名:密码@cluster.mongodb.net/crypto-wallet-explorer
REDIS_URL=redis://用户名:密码@redis-cloud-url:port
REDIS_ENABLED=true
ADMIN_SETUP_KEY=your-secure-key
NEXT_PUBLIC_APP_URL=https://your-app.vercel.app
NEXT_PUBLIC_BASE_URL=https://your-app.vercel.app
NODE_ENV=production
```

3. 部署

### 注意事项
- Vercel 有函数执行时间限制（10秒）
- 不适合长时间运行的任务
- 建议使用 Docker 或 PM2 部署

### 3. 数据库备份

```bash
# MongoDB 备份
mongodump --uri="mongodb://用户名:密码@localhost:27017/crypto-wallet-explorer" --out=/backup/$(date +%Y%m%d)

# 自动备份脚本
cat > /usr/local/bin/backup-mongo.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/backup/mongodb"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
mongodump --uri="$MONGODB_URI" --out=$BACKUP_DIR/$DATE
find $BACKUP_DIR -type d -mtime +7 -exec rm -rf {} +
EOF

chmod +x /usr/local/bin/backup-mongo.sh

# 添加到 crontab（每天凌晨 2 点备份）
0 2 * * * /usr/local/bin/backup-mongo.sh
```

## 故障排查

### 问题 1: 应用无法启动

```bash
# 检查日志
docker-compose logs app
# 或
pm2 logs crypto-wallet-explorer

# 检查端口占用
netstat -tulpn | grep 3000

# 检查环境变量
docker-compose exec app env | grep MONGODB
```

### 问题 2: 数据库连接失败

```bash
# 测试 MongoDB 连接
mongo "mongodb://用户名:密码@localhost:27017/crypto-wallet-explorer?authSource=admin"

# 检查 MongoDB 状态
sudo systemctl status mongod

# 查看 MongoDB 日志
sudo tail -f /var/log/mongodb/mongod.log
```

### 问题 3: Redis 连接失败

```bash
# 测试 Redis 连接
redis-cli ping

# 检查 Redis 状态
sudo systemctl status redis

# 查看 Redis 日志
sudo tail -f /var/log/redis/redis-server.log
```

### 问题 4: 内存不足

```bash
# 增加 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=4096" npm start

# 或在 PM2 配置中
max_memory_restart: '2G'
```
