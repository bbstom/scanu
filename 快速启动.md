# 快速启动指南

## 三步启动

### 1. 安装依赖
```bash
npm install
```

### 2. 配置环境变量
创建 `.env` 文件：
```bash
# 复制示例文件
cp .env.example .env
```

编辑 `.env` 文件，配置你的数据库地址和 API Keys：
```env
# MongoDB（必需 - 根据你的实际情况选择）

# 1. 无认证（本地开发）
MONGODB_URI=mongodb://localhost:27017/crypto-wallet-explorer

# 2. 有认证（推荐）
# MONGODB_URI=mongodb://用户名:密码@localhost:27017/crypto-wallet-explorer?authSource=admin

# 3. 远程数据库
# MONGODB_URI=mongodb://用户名:密码@服务器IP:27017/crypto-wallet-explorer?authSource=admin

# 4. MongoDB Atlas（云数据库）
# MONGODB_URI=mongodb+srv://用户名:密码@cluster.mongodb.net/crypto-wallet-explorer?retryWrites=true&w=majority

# Redis（可选）
REDIS_URL=redis://localhost:6379

# API Keys（可选 - 可以在后台管理界面配置）
# ETHERSCAN_API_KEY=你的etherscan_api_key
# TRONGRID_API_KEY=你的trongrid_api_key

# 应用配置
NEXT_PUBLIC_APP_URL=http://localhost:3000
NODE_ENV=development
```

**重要提示**：
- API Keys 现在可以在后台管理界面配置，无需在 .env 中设置
- 如果密码包含特殊字符（如 @, :, / 等），需要进行 URL 编码
- 详细配置说明请查看 [MongoDB配置说明.md](./MongoDB配置说明.md)

### 3. 启动开发服务器
```bash
npm run dev
```

访问：http://localhost:3000

## 数据库配置说明

### MongoDB 配置选项

#### 选项 1：无认证（本地开发）
```env
MONGODB_URI=mongodb://localhost:27017/crypto-wallet-explorer
```
适用于本地开发，MongoDB 未设置密码

#### 选项 2：有认证（推荐）
```env
MONGODB_URI=mongodb://用户名:密码@localhost:27017/crypto-wallet-explorer?authSource=admin
```
示例：
```env
MONGODB_URI=mongodb://admin:mypassword@localhost:27017/crypto-wallet-explorer?authSource=admin
```

#### 选项 3：远程数据库
```env
MONGODB_URI=mongodb://用户名:密码@服务器IP:27017/crypto-wallet-explorer?authSource=admin
```
示例：
```env
MONGODB_URI=mongodb://dbuser:dbpass@192.168.1.100:27017/crypto-wallet-explorer?authSource=admin
```

#### 选项 4：MongoDB Atlas（云数据库，推荐）
```env
MONGODB_URI=mongodb+srv://用户名:密码@cluster.mongodb.net/crypto-wallet-explorer?retryWrites=true&w=majority
```
示例：
```env
MONGODB_URI=mongodb+srv://myuser:mypass@cluster0.abc123.mongodb.net/crypto-wallet-explorer?retryWrites=true&w=majority
```

### 特殊字符处理

如果密码包含特殊字符，需要进行 URL 编码：
- `@` → `%40`
- `:` → `%3A`
- `/` → `%2F`
- `?` → `%3F`

示例：密码 `pass@word` 应写为 `pass%40word`

详细说明请查看：[MongoDB配置说明.md](./MongoDB配置说明.md)

### Redis 配置（可选）

如果你有 Redis 服务，可以配置缓存：
```env
REDIS_URL=redis://你的redis地址:6379
```

如果没有 Redis，项目仍然可以正常运行，只是没有缓存功能。

## 获取 API Keys

### Etherscan API Key（必需）
1. 访问：https://etherscan.io/
2. 注册并登录
3. 进入：https://etherscan.io/myapikey
4. 创建免费 API Key

### TronGrid API Key（必需）
1. 访问：https://www.trongrid.io/
2. 注册账号
3. 在控制台创建 API Key

### CoinGecko API（可选）
- 免费版无需 API Key
- 如需更高限制：https://www.coingecko.com/en/api

## 常见问题

### Q: npm install 失败
A: 尝试清除缓存：
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### Q: MongoDB 连接失败
A: 检查以下几点：
1. 确保 MONGODB_URI 配置正确
2. 检查数据库服务是否运行
3. 检查网络连接和防火墙设置
4. 如果使用 MongoDB Atlas，检查 IP 白名单
5. 查看终端错误日志获取详细信息

### Q: API 查询失败
A: 检查：
1. API Key 是否正确配置
2. 网络连接是否正常
3. API 请求限制是否超出

### Q: 页面显示错误
A: 查看浏览器控制台和终端日志，通常会有详细错误信息

## 生产部署

### Vercel（推荐）
1. 推送代码到 GitHub
2. 在 Vercel 导入项目
3. 配置环境变量（包括 MONGODB_URI）
4. 自动部署

注意：Vercel 部署时，确保 MongoDB 数据库可以从外网访问，或使用 MongoDB Atlas。

### 自建服务器
```bash
# 构建
npm run build

# 启动
npm start
```

### 环境变量配置
生产环境需要配置：
- MONGODB_URI（必需）
- ETHERSCAN_API_KEY（必需）
- TRONGRID_API_KEY（必需）
- REDIS_URL（可选）
- NEXT_PUBLIC_APP_URL（必需）
