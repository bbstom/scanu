# 后台管理系统使用说明

## 功能说明

现在 API Keys 可以通过后台管理界面配置，无需修改 `.env` 文件。

## 使用步骤

### 1. 首次使用 - 创建管理员账号

访问：`http://localhost:3000/admin/setup`

- 输入用户名（至少3个字符）
- 输入密码（至少6个字符）
- 确认密码
- 点击"创建管理员"

注意：只能创建一个管理员账号，创建后此页面将不可用。

### 2. 登录后台

访问：`http://localhost:3000/admin`

- 输入用户名和密码
- 点击"登录"

### 3. 配置 API Keys

登录成功后会自动跳转到设置页面：`http://localhost:3000/admin/settings`

可以配置以下 API Keys：

#### Etherscan API Key（必需）
- **用途**：查询以太坊钱包余额和交易记录
- **获取方式**：
  1. 访问 https://etherscan.io/
  2. 注册并登录
  3. 进入 https://etherscan.io/myapikey
  4. 点击 "Add" 创建新的 API Key
  5. 复制 API Key 粘贴到后台

#### TronGrid API Key（必需）
- **用途**：查询波场钱包余额和 USDT-TRC20
- **获取方式**：
  1. 访问 https://www.trongrid.io/
  2. 注册账号
  3. 在控制台创建 API Key
  4. 复制 API Key 粘贴到后台

#### CoinGecko API Key（可选）
- **用途**：查询加密货币价格
- **说明**：免费版无需 API Key，留空即可
- **获取方式**：如需更高限制，访问 https://www.coingecko.com/en/api

### 4. 保存设置

- 填写完 API Keys 后，点击"保存设置"
- 看到"保存成功"提示即可
- 设置会立即生效

### 5. 退出登录

点击右上角"退出登录"按钮

## 配置优先级

系统会按以下优先级读取配置：

1. **环境变量**（.env 文件）- 最高优先级
2. **数据库配置**（后台管理设置）- 次优先级
3. **默认值**（空字符串）- 最低优先级

这意味着：
- 如果 `.env` 文件中有配置，会优先使用 `.env` 的配置
- 如果 `.env` 中没有，则使用后台管理配置的值
- 如果都没有，则使用空字符串

## 安全建议

### 生产环境

1. **修改密码哈希方式**
   - 当前使用简单的 Base64 编码
   - 生产环境应使用 bcrypt 或其他安全哈希算法

2. **使用 JWT Token**
   - 当前使用简单的 Base64 token
   - 生产环境应使用 JWT 并设置过期时间

3. **添加 HTTPS**
   - 确保后台管理页面使用 HTTPS
   - 防止密码和 API Key 被窃取

4. **限制访问**
   - 可以添加 IP 白名单
   - 或使用 VPN 访问后台

### 推荐改进

```typescript
// 安装 bcrypt
npm install bcrypt
npm install --save-dev @types/bcrypt

// 使用 bcrypt 哈希密码
import bcrypt from 'bcrypt';

async function hashPassword(password: string): Promise<string> {
  return await bcrypt.hash(password, 10);
}

async function verifyPassword(password: string, hash: string): Promise<boolean> {
  return await bcrypt.compare(password, hash);
}
```

```javascript

```

## 常见问题

### Q: 忘记管理员密码怎么办？
A: 需要直接操作数据库删除管理员记录，然后重新创建：
```bash
# 连接 MongoDB
mongo

# 使用数据库
use crypto-wallet-explorer

# 删除管理员
db.admins.deleteMany({})

# 然后访问 /admin/setup 重新创建
```

### Q: 修改 API Key 后需要重启服务吗？
A: 不需要。设置会立即生效（有1分钟缓存）。

### Q: 可以配置多个管理员吗？
A: 当前版本只支持一个管理员。如需多个管理员，需要修改代码。

### Q: API Key 存储安全吗？
A: API Key 以明文存储在数据库中。建议：
- 使用数据库加密
- 限制数据库访问权限
- 定期更换 API Key

## 页面路由

- `/admin/setup` - 创建管理员（仅首次）
- `/admin` - 管理员登录
- `/admin/settings` - 配置管理（需登录）

## 技术实现

### 前端
- React + Next.js 14
- 客户端组件（"use client"）
- localStorage 存储 token

### 后端
- Next.js API Routes
- MongoDB 存储配置
- 简单的 token 认证

### 数据流
1. 用户在后台配置 API Key
2. 保存到 MongoDB Settings 集合
3. 查询时从数据库读取（带缓存）
4. 如果环境变量存在，优先使用环境变量
