# 管理后台安全配置指南

## 安全问题说明

### 原有问题
在之前的版本中，任何人都可以访问 `/admin/setup` 创建管理员账号，存在严重的安全隐患。

### 解决方案
现在添加了以下安全措施：
1. ✅ **初始化密钥保护**：需要提供正确的密钥才能创建管理员
2. ✅ **单管理员限制**：系统只允许创建一个管理员账号
3. ✅ **环境变量配置**：密钥存储在服务器端，不会泄露

## 配置步骤

### 步骤 1: 设置初始化密钥

在 `.env` 文件中添加：

```env
# 管理员初始化密钥（重要！生产环境必须设置）
ADMIN_SETUP_KEY=your-secret-setup-key-here
```

**密钥要求**：
- 使用强密码（建议 20+ 字符）
- 包含大小写字母、数字和特殊字符
- 不要使用常见词汇
- 妥善保管，不要泄露

**示例密钥**：
```env
ADMIN_SETUP_KEY=Kx9#mP2$vL8@nQ5!wR7^tY4&uI3*oP6
```

### 步骤 2: 重启服务器

修改 `.env` 文件后，需要重启服务器：

```bash
# 停止服务器 (Ctrl+C)
npm run dev
```

### 步骤 3: 创建管理员账号

1. 访问：`http://localhost:3000/admin/setup`
2. 填写信息：
   - 用户名：例如 `admin`
   - 密码：例如 `Admin@123456`
   - 确认密码：再次输入密码
   - **初始化密钥**：输入你在 `.env` 中设置的 `ADMIN_SETUP_KEY`
3. 点击"创建管理员"

## 安全级别

### 开发环境（不推荐用于生产）

如果不设置 `ADMIN_SETUP_KEY`，任何人都可以创建管理员（仅限首次）：

```env
# 留空或不设置
# ADMIN_SETUP_KEY=
```

⚠️ **警告**：这种配置仅适用于本地开发环境，绝不要用于生产环境！

### 生产环境（推荐）

**必须设置强密钥**：

```env
# 使用强密码
ADMIN_SETUP_KEY=Kx9#mP2$vL8@nQ5!wR7^tY4&uI3*oP6
```

✅ **优点**：
- 只有知道密钥的人才能创建管理员
- 防止未授权访问
- 符合安全最佳实践

## 生成强密钥

### 方法 1: 使用 Node.js

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

### 方法 2: 使用 OpenSSL

```bash
openssl rand -base64 32
```

### 方法 3: 在线生成器

访问：https://www.random.org/passwords/

设置：
- 长度：32
- 包含：大小写字母、数字、符号

## 额外安全措施

### 1. 限制 IP 访问

在生产环境中，建议通过防火墙或 Nginx 限制管理后台的访问 IP：

```nginx
# Nginx 配置示例
location /admin {
    allow 192.168.1.100;  # 允许的 IP
    deny all;              # 拒绝其他所有 IP
    proxy_pass http://localhost:3000;
}
```

### 2. 使用 HTTPS

生产环境必须使用 HTTPS：

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:3000;
    }
}
```

### 3. 设置强密码策略

管理员密码建议：
- 至少 12 个字符
- 包含大小写字母、数字和特殊字符
- 不使用常见密码
- 定期更换（每 3 个月）

### 4. 启用日志记录

监控管理后台的访问日志：

```bash
# 查看 Nginx 访问日志
tail -f /var/log/nginx/access.log | grep /admin
```

### 5. 定期备份数据库

```bash
# MongoDB 备份
mongodump --uri="mongodb://user:pass@host:27017/crypto-wallet-explorer" --out=/backup/$(date +%Y%m%d)
```

## 常见问题

### Q: 忘记了初始化密钥怎么办？

A: 查看服务器的 `.env` 文件：

```bash
cat .env | grep ADMIN_SETUP_KEY
```

### Q: 如何重置管理员账号？

A: 删除数据库中的管理员记录：

```bash
# 连接 MongoDB
mongo mongodb://root:Ubuntu123!@172.16.254.100:27017/crypto-wallet-explorer?authSource=admin

# 删除管理员
db.admins.deleteMany({})

# 退出
exit
```

然后重新访问 `/admin/setup` 创建新账号。

### Q: 可以修改初始化密钥吗？

A: 可以。修改 `.env` 文件中的 `ADMIN_SETUP_KEY`，然后重启服务器。但这不会影响已创建的管理员账号。

### Q: 初始化密钥会过期吗？

A: 不会。初始化密钥只在创建管理员时使用，创建后就不再需要。

### Q: 如何禁用 setup 页面？

A: 一旦创建了管理员，setup 页面会自动禁用（显示"管理员已存在"）。如果想完全禁用，可以删除 `src/app/admin/setup` 目录。
