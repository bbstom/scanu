# 构建问题已解决 ✅

## 问题原因

你遇到的 `SIGBUS` 错误是因为：
- 原配置要求 4GB 内存（`--max-old-space-size=4096`）
- 服务器内存不足，导致构建崩溃

## 现在如何部署？

### 方法 1：一键部署（最简单）

在宝塔面板终端执行：

```bash
cd /www/wwwroot/scanu.cc/scanu
bash 一键部署.sh
```

### 方法 2：手动部署（3条命令）

```bash
cd /www/wwwroot/scanu.cc/scanu
rm -rf .next
NODE_OPTIONS='--max-old-space-size=512' npm run build
```

### 方法 3：使用完整部署脚本

```bash
cd /www/wwwroot/scanu.cc/scanu
bash deploy-production.sh
```

## 如果还是内存不足？

### 选项 A：使用更小的内存

```bash
NODE_OPTIONS='--max-old-space-size=256' npm run build
```

### 选项 B：在本地构建

1. 在你的电脑上：
```bash
npm install
npm run build
```

2. 打包 .next 目录：
```bash
tar -czf build.tar.gz .next
```

3. 上传到服务器并解压

4. 在服务器启动：
```bash
npm ci --production
npm start
```

## 启动应用

构建成功后，选择一种方式启动：

2. 添加项目

## 常见问题

### Q: 构建需要多长时间？
A: 通常 3-10 分钟，取决于服务器性能。

### Q: 需要多少内存？
A: 
- 最低：512MB RAM + 2GB Swap
- 推荐：1GB RAM + 1GB Swap
- 理想：2GB RAM

### Q: 如何查看构建进度？
A: 构建过程会实时显示进度，包括编译的页面数量。

### Q: 构建失败怎么办？
A: 
1. 检查内存：`free -h`
2. 创建 Swap：见 `宝塔部署步骤.md`
3. 使用更小内存：`NODE_OPTIONS='--max-old-space-size=256' npm run build`
4. 在本地构建后上传
